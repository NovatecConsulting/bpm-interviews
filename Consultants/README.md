# Consultants
Dieses Projekt ist für Interviews mit Interessenten auf eine Anstellung zum Consultant ausgelegt

# Aufgaben

## Grundlagen Betrieb Camunda Platform
1. Klone das Repository und importiere das Projekt in deine Entwicklungsumgebung
2. Erkläre, wie die vorliegende Form des Betriebs von Camunda Platform bezeichnet wird
3. Nenne zwei weitere Formen des Betriebs von Camunda Platform
4. Erläutere Vor- und Nachteile der verschiedenen Betriebsformen von Camunda Platform

## Grundlagen Prozessmanagement
1. Öffne die Datei 'application.bpmn' mit dem Camunda Modeler
2. Erkläre mit Worten das dargestellte Prozessmodell
3. Benenne die im Prozessmodell dargestellten Flusselement-Typen und erkläre ihre Funktion
4. Erkläre, ob es sich um ein strategisches, operatives oder technisches Prozessmodell handelt und worin die Unterschiede zwischen diesen Prozessmodell-Arten liegen
5. Nenne drei Aspekte, die an diesem Prozessmodell verbessert werden können

## Modellierung
1. Ergänze das Modell dahingehend, dass im Falle einer unvollständigeng Bewerbung anstatt einer direkten Absage eine Nachricht an den Bewerber mit Bitte um Vervollständigung der Unterlagen versandt wird. Daraufhin soll auf den erneuten Eingang von Bewerbungsunterlagen gewartet werden, die dann wie die initialen Unterlagen geprüft werden sollen.
2. Ergänze das Modell so, dass nach 14 Tagen, in denen keine korrigierten Bewerbungsunterlagen eingegangen sind, dem Bewerber abgesagt wird.
3. Ergänze die technischen Aspekte des Prozessmodells. Erstelle dafür die Implementierung für den Send Task selbst

## Testing
1. Öffne den Unit-Test für dieses Prozessmodell
2. Behebe den fehlschlagenden Test
3. Stelle die Testklasse so ein, dass sie eine vollständige Abdeckung (100%) des Prozessmodells erfordert und andernfalls fehlschlägt
4. Ergänze geeignete Tests, um 100% Testabdeckung zu erreichen

## Java-und Rest-Integration
1. Starte die Anwendung und navigiere auf die Seite des Swagger-UIs
2. Erkläre im groben die Funktionalität von Swagger und teste die vorgefertigten Endpunkte
3. Ergänze die Implementierungen zu den vorgefertigten Endpunkten und verifiziere ihre Funktionalität
